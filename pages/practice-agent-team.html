5-Agent 团队架构

基于 OpenClaw 构建的多 Agent 协作系统，5个AI Agent 各司其职、协同运转。

团队架构

团队由5个独立的AI Agent组成，每个Agent拥有独立的SOUL.md、MEMORY.md和技能体系，通过飞书群协作。

特别助理

职责：中央协调，统筹全局

  • 发现和转交需要整理的信息
  • 每日09:00执行项目进度检查
  • 协调各Agent之间的任务分配
  • 向老板汇报项目进展

HR大姐头

职责：招聘和管理Agent

  • 评估新Agent的需求和能力模型
  • 创建和配置Agent的workspace
  • 定期复盘团队表现

知识管家

职责：信息整理与观点库维护

  • 接收并结构化处理信息
  • 维护知识库和观点库
  • 编译LLM Wiki内容
  • 每日向PR推送可公开的高价值观点

妙趣AI

职责：miaoquai.com 网站运营

  • RSS聚合定时任务管理
  • GitHub Discussions 技术营销
  • 多平台内容分发
  • 社区互动和数据追踪

PR

职责：个人品牌建设与内容运营

  • 将知识库观点转化为可发布文章
  • 小红书、公众号等内容运营
  • Build in Public风格品牌输出

协作机制

飞书群协作

每个Agent绑定独立的飞书群，通过OpenClaw的sessions系统实现跨Agent通信。

MEMORY同步强制机制

每日沟通后必须更新各自的MEMORY.md文件，确保Agent之间信息同步。

边界清晰

每个Agent的SOUL.md中明确规定：

  • 职责范围：只做自己分内的事
  • 禁止事项：不越权处理其他Agent的工作
  • 路由规则：遇到不属于自己职责的任务，转发给正确的Agent

技能矩阵

团队共配置30+个跨平台技能，覆盖：

  • 内容创作（AI去味、SEO优化、广告创意）
  • 社媒运营（小红书、微信公众号、LinkedIn）
  • 知识管理（LLM知识库、记忆系统）
  • 开发工具（前端设计、浏览器自动化）
  • 商业分析（竞争分析、关键词研究、营销心理学）

关键洞察

"多Agent协作的核心不是'让一个AI更聪明'，而是'让多个AI各司其职'。边界清晰、职责分明，才能避免冲突和重复劳动。"

Agent的workspace必须干净，别人的工作记录会污染身份认知（2026-03-16教训）。
